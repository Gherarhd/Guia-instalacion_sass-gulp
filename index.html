Instrucciones de Instalación para Node.js y Gulp:

Paso 1: Instala Node.js

Asegúrate de tener Node.js instalado en tu sistema. Puedes descargar la última versión desde el sitio web oficial: Node.js

Paso 2:Verifica la instalación de Node.js, npm y npx

Abre tu terminal y ejecuta los siguientes comandos para asegurarte de que Node.js, npm y npx estén instalados correctamente:

Este comando mostrará la versión de Node.js instalada.

node -v

Este comando mostrará la versión de NPM instalada.

npm -v

Este comando mostrará la versión de NPX instalada.

npx -v

Paso 3: Instala Gulp de forma global

Utiliza el siguiente comando para instalar Gulp globalmente en tu sistema:

npm install --global gulp-cli

Paso 4: Verifica la instalación de Gulp

Confirma que Gulp se ha instalado correctamente ejecutando el siguiente comando:

gulp -v

Paso 5: Inicia npm en tu Proyecto y Verifica tu archivo package.json

Después de haber instalado Node.js y Gulp, es esencial iniciar npm en tu proyecto para gestionar las dependencias. Asegúrate de estar en el directorio de tu proyecto y ejecuta los siguientes comandos en tu terminal:

npm init

Ahora, abre tu archivo package.json y asegúrate de que contenga la información correcta. Un ejemplo básico podría ser similar al siguiente:

Asegúrate de personalizar la información según las necesidades de tu proyecto.

Este paso es crucial porque Gulp y otras herramientas a menudo se integran con npm para gestionar las tareas del proyecto. Al tener un archivo package.json correctamente configurado, garantizas que las dependencias se manejen de manera eficiente.

Paso 6: Instala Gulp Localmente y Verifica el Archivo package-lock.json

Para instalar Gulp localmente en tu proyecto y asegurarte de que se refleje en el archivo package-lock.json , sigue estos pasos:

npm install --save-dev gulp

Este comando instalará Gulp como una dependencia de desarrollo (devDependencies) y actualizará tu archivo package.json y package-lock.json automáticamente.

Asegúrate de que tu archivo package.json ahora incluya la sección devDependencies con Gulp:

Este paso asegura que Gulp esté instalado localmente en tu proyecto y que las versiones exactas se registren en el archivo package-lock.json. Además, la carpeta node_modules debería haber sido creada en tu directorio de proyecto.

Finalmente, verifica que la instalación de Gulp haya sido exitosa ejecutando:

gulp -v

Este comando mostrará la versión de Gulp que se ha instalado localmente en tu proyecto. Ahora estás listo para comenzar a configurar y utilizar Gulp en tu entorno de desarrollo

Añadiendo Watch y Otros Plugins

1. Funciones y Plugins Adicionales

La función `watch` integrada en Gulp se utiliza para observar cambios en archivos específicos y ejecutar tareas automáticamente cuando se detectan modificaciones. En el contexto de tu proyecto, esta función se emplea para monitorizar cambios en el archivo SASS (`app.scss`) y ejecutar automáticamente la tarea de compilación de CSS (`css`) al guardar modificaciones.

gulp-plumber es un middleware de Gulp que evita que la ejecución de la tarea se detenga cuando ocurren errores. En lugar de salir abruptamente, plumber captura los errores y permite que la ejecución de Gulp continúe, proporcionando información sobre los errores en la consola.

npm i -D gulp-plumber

gulp-postcss es un plugin que permite el procesamiento de CSS con varias herramientas, en lugar de solo SASS. Se utiliza para aplicar una serie de transformaciones al CSS después de ser compilado, y se puede combinar con diferentes plugins para realizar tareas específicas, como autoprefijado.

Este plugin de PostCSS, específicamente autoprefixer, se utiliza para agregar automáticamente prefijos de proveedores a las propiedades CSS. Ayuda a garantizar la compatibilidad con navegadores antiguos al agregar automáticamente los prefijos necesarios para las reglas CSS que requieren prefijos de proveedores diferentes.

npm i --save-dev autoprefixer gulp-postcss
